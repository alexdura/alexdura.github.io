Alexandru Dura

I am a compiler engineer with a PhD in static program analysis.

I did my PhD at Lund University, under the supervision of Christoph Reichenbach and Emma Söderberg. In my thesis, I explore declarative techniques for building static code checkers that have comparable performance and analysis quality with state-of-the-practice tools. The aim of this work is to reduce the overhead of defining bug detectors, and thus enable more software developers to customize static code checkers for the needs of their projects.

Before this, I developed compilers for Arm (2012-2018) and Movidius (2010-2012). I contributed to all stages of a compiler pipeline, from machine code generation for highly specialized architectures (GPUs and VPUs) to graphics-specific language frontends and optimizations.

Interests

Static Program Analysis, Datalog, Compiler Construction, Programming Language Design

Projects

  • 2022 - Clog

    • Clog is a fully declarative static code checking framework for C. It combines Datalog reasoning with syntactic pattern matching in Clang to enable the definition of bug detectors for C, in a manner that is independent of any representation of programs, other than source code.

  • 2020 - JavaDL

    • JavaDL is a programming language and a runtime for defining static bug checkers for Java. It is an extension of Datalog with syntactic patterns and semantic predicates. The runtime supports alternative evaluation modes: incremental and exhaustive.

  • 2019 - MetaDL

    • MetaDL is an extension of the Datalog language with syntactic patterns, to enable the analysis of Datalog programs inside Datalog. We are currently exploring ways to use the pattern syntax to analyze other programming languages and to express code transformations.
